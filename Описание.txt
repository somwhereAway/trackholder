Оприсание.
	* Телеграм бот для хранения данных в формате kml
Функционал.
	* При получении документа:
		-считает хэш сумму
		-проверяет в БД
			-если есть
				-то отвечает что такя есть.
				-показывает Автора записи.
				еслим пользователь автор
					меню редактирования авторских тэгов
					меню редактирования тематических тэгов
				
			-если нет
				-сообщение :"таакого файла нет в БД" +
					-кнопка зписи без тэга
					-кнопка записи тэгов
						-авторские тэги
						-пользовательские тэги
				-записывает на диск.
				-генерирует название их хэш
				-записывает в БД
				-Пишет сообщение, что сохранено.
	* команда /my_merged
		- создает обьединенный файл из файлов ползователя
		- проверяет размер файла
			- если больше 500 мб отправляет уведомление.
			- если меньше, оправляет файл
	* команда /all_merged (доступна только суперпользователям)
		-создает обьедененный файл из всех файлов
		-проверяет размер файла
			- если больше 500 мб отправляет уведомление.
			- если меньше, оправляет файл
	* команда /search_object 
		виды поиска
			-точный поиск по названию
			-неточный поиск по названию
			-поиск по описанию
		возвращает список подходящих обьектов в виде кнопок
		при выборе
		Если суперпользователь:
			 -отправляет kml с объектом
		-палитиру с возможностью открыть страничку обьекта. (База знаний speleo19)
		
	* Страничка админа для создания записи об обьекте в БД
